# 아이템 인벤토리 시스템 변경 로그

## 📅 변경 날짜: 2026-05-13

## 🔄 변경 개요

아이템 시스템을 **1회 구매 사용** 방식에서 **소지 개수 기반 인벤토리** 방식으로 전환

---

## 🆚 변경 전 vs 변경 후

### 이전 시스템 (1회 구매 사용)

```
┌─────────────────────────────────────┐
│ 아이템: 🛡️ 보호 방패 (150G)        │
│                                     │
│ 상태:                               │
│ - 미사용: 클릭하여 골드로 구매      │
│ - 사용됨: 회색 처리, 재구매 불가    │
│                                     │
│ 문제점:                             │
│ - 1판당 각 아이템 1회만 사용 가능   │
│ - 전략적 선택 폭 제한               │
│ - 반복 사용 불가                    │
└─────────────────────────────────────┘
```

### 변경 후 시스템 (소지 개수 인벤토리)

```
┌─────────────────────────────────────┐
│ 아이템: 🛡️ 보호 방패               │
│ 소지: 2개          [구매: 150G]     │
│                                     │
│ 동작:                               │
│ - 소지 2개 → 클릭 시 즉시 사용      │
│ - 소지 0개 → 클릭 시 골드로 구매    │
│                                     │
│ 장점:                               │
│ ✅ 초기 아이템 지급 (각 2개)        │
│ ✅ 소진 시 골드로 재구매 가능       │
│ ✅ 전략적 재고 관리 가능            │
└─────────────────────────────────────┘
```

## 💻 주요 코드 변경사항

### 1. 변수 변경

#### Before
```javascript
let usedItems = [];  // 사용한 아이템 ID 배열
```

#### After
```javascript
let itemInventory = {};  // 아이템 소지 개수 { 'shield': 2, 'potion': 2, ... }
```

### 2. 초기화 로직

#### Before
```javascript
function startGame() {
    // ...
    usedItems = [];
    // ...
}
```

#### After
```javascript
function startGame() {
    // ...
    // 초기 아이템 지급 (각 2개씩)
    itemInventory = {};
    ITEMS.forEach(item => {
        itemInventory[item.id] = 2;
    });
    // ...
}
```

### 3. 렌더링 로직

#### Before
```javascript
function renderItems() {
    ITEMS.forEach(item => {
        const isUsed = usedItems.includes(item.id);
        const cantAfford = playerGold < item.cost;
        
        let className = 'item-slot';
        if (isUsed) className += ' used';  // 사용됨 = 회색 처리
        else if (cantAfford) className += ' disabled';
        
        slot.innerHTML = `
            ${item.icon}
            <span class="item-cost">${item.cost}G</span>
        `;
    });
}
```

#### After
```javascript
function renderItems() {
    ITEMS.forEach(item => {
        const ownedCount = itemInventory[item.id] || 0;
        const cantAfford = ownedCount === 0 && playerGold < item.cost;
        
        let className = 'item-slot';
        if (ownedCount === 0 && cantAfford) className += ' disabled';
        
        slot.innerHTML = `
            ${item.icon}
            <span class="item-count">${ownedCount}</span>
            <span class="item-cost">${ownedCount === 0 ? item.cost + 'G' : '사용'}</span>
            <span class="item-tooltip">
                소지: ${ownedCount}개<br>
                ${ownedCount === 0 ? '구매 비용: ' + item.cost + 'G' : '클릭하여 사용'}
            </span>
        `;
    });
}
```

### 4. 사용 로직

#### Before
```javascript
function useItem(item) {
    if (playerGold < item.cost) return;
    if (usedItems.includes(item.id)) return;  // 이미 사용한 아이템은 차단
    
    playerGold -= item.cost;  // 골드 차감
    usedItems.push(item.id);  // 사용 목록에 추가
    // 효과 적용
}
```

#### After
```javascript
function useItem(item) {
    const ownedCount = itemInventory[item.id] || 0;
    
    // 소지 개수가 0이면 골드로 구매
    if (ownedCount === 0) {
        if (playerGold < item.cost) return;
        playerGold -= item.cost;
        itemInventory[item.id] = 1;
        return;
    }
    
    // 소지 개수가 있으면 사용 (골드 차감 없음)
    itemInventory[item.id]--;  // 개수 차감
    // 효과 적용
}
```

#### Before
```
┌──────────┐
│   🛡️    │  ← 아이콘만
│  150G    │  ← 가격
└──────────┘
```

#### After
```
┌──────────┐

└──────────┘
```

#### Before
```

```

#### After
```
보호 방패
이번 턴 패배해도 HP 손실 없음
소지: 2개
클릭하여 사용
```

## 🎮 게임플레이 변화

### 초기 상태
- **이전**: 모든 아이템 미보유 (골드로만 구매)
- **현재**: 모든 아이템 2개씩 지급 (총 12개)

### 아이템 사용 흐름

#### 시나리오 1: 소지 개수가 있을 때
```
1. 플레이어가 🛡️ 보호 방패 클릭
2. 소지 개수: 2개 → 1개로 감소
3. 골드 변화 없음
4. 효과 즉시 활성화
5. UI 업데이트 (개수 표시: 1개)
```

#### 시나리오 2: 소지 개수가 0일 때
```
1. 플레이어가 🛡️ 보호 방패 클릭
2. 소지 개수: 0개 확인
3. 골드 150G 차감
4. 소지 개수: 0개 → 1개로 증가
5. 메시지: "🛡️ 보호 방패 구매!"
6. UI 업데이트 (개수 표시: 1개)
7. 다시 클릭하면 사용됨
```

## 🎯 전략적 영향

### 1. 초반 전략 다양화
- **이전**: 골드가 부족해 아이템 사용 꺼림
- **현재**: 초기 아이템 2개로 적극적 사용 가능

### 2. 자원 관리 중요성 증가
- 아이템 소지 개수 관리 필요
- 언제 사용하고 언제 재구매할지 판단

### 3. 후반 역전 가능성
- 골드가 부족해도 보유 아이템으로 버티기 가능
- 전략적 아이템 비축 가능

### 4. 위험-보상 밸런스
- 초기 아이템 2개 사용 후 재구매 비용 발생
- 아껴 쓸지 적극적으로 쓸지 선택의 폭 확대

## ⚖️ 밸런스 영향 분석

### 게임 난이도
- **이전 난이도**: 중간 (아이템 1회 제한)
- **현재 난이도**: 쉬움 → 중간 (초기 아이템 지급으로 초반 쉬워짐)

### 게임 길이
- **이전**: 평균 10턴
- **현재**: 평균 12-15턴 (초기 아이템으로 생존력 증가)

### 골드 소진율
- **이전**: 배당 중심 (700G/5판)
- **현재**: 배당 + 아이템 재구매 (700G + 300G = 1,000G/5판)
- **증가율**: +43%

## 💰 수익화 영향

### 긍정적 영향
1. **초기 진입 장벽 낮춤**: 신규 유저가 아이템을 체험 가능
2. **아이템 가치 인지**: 2개 사용 후 효용성 체감 → 재구매 유도
3. **골드 소진 증가**: 아이템 재구매로 골드 소모 43% 증가

### 고려 사항
1. **초기 자산 증가**: 1,000G → 1,750G 상당 (밸런스 조정 필요)
2. **게임 길이 증가**: 턴 수 증가 → 세션 시간 증가 (광고 노출 기회 증가)
3. **아이템 가치 하락 우려**: 초기 무료 지급으로 과금 유인 감소 가능

### 대응 방안
1. AI 난이도 상향 조정
2. 초기 지급 개수 조정 (2개 → 1개)
3. 아이템 가격 재조정
4. 프리미엄 아이템 추가 (다이아 전용)

## 📝 개발자 노트

**변경 이유**:
- 1회 사용 제한으로 인한 전략적 선택 폭 제약
- 아이템 체험 기회 부족으로 가치 인지 저하
- 반복 플레이 시 단조로움

**기대 효과**:
- 초기 아이템 체험 → 가치 인지 → 재구매 유도
- 자원 관리 전략 추가 (인벤토리 관리)
- 후반 역전 가능성 증가로 재미 향상

**주의 사항**:
- 초기 자산 증가로 인한 난이도 하락 → AI 조정 필요
- 아이템 재구매 비용이 골드 순환에 미치는 영향 모니터링
- 신규 유저와 기존 유저 간 밸런스 유지
